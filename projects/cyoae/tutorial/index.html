How to write an adventure

Each time the user has to make a decision and is shown where they are right now is called a state. Each state is identified by a unique id (which is text that can contain spaces).

Once you have planned out your story and know how you will use states, create a .js file on your computer and open it in any plain text editor.

Begin writing a state like this:

state["wake up"] = {
};

Magenta text represents state ids.

Within the curly brackets is the information for each state. Each state has four traits: the title, the image or gif, the description paragraph, and the set of choices the user can make. The first three are the easiest to make, like so:

state["wake up"] = {
  title: "You woke up.",
  desc: "Good morning!",
  image: "photo.png"
};

The yellow text represents the traits of the state, and the cyan text represents text.

In the example above, when the user gets to this page, the title "You woke up." will be at the top, an image of photo.png will show, and a paragraph below will say "Good morning!".

No matter how many traits you include, all except the last one will need a comma after it.

The final trait, the options for the user, is more complicated. It starts off like the others, but instead of using text surrounded by quotes, you create a set of options with brackets.

state["wake up"] = {
  title: "You woke up.",
  desc: "Good morning!",
  image: "photo.png",
  option: [
  ]
};

Within the square brackets, put each option in curly brackets with two traits: label and link.

state["wake up"] = {
  title: "You woke up.",
  desc: "Good morning!",
  image: "photo.png",
  option: [
    {label: "Go to store", link: "store"},
    {label: "Go to school", link: "school"}
  ]
};

In the example above, the user has two options: "Go to store", which links them to state "store", or "Go to school", which links them to state "school". Like before, all but the last option need a comma after the curly bracket. In the end, this is how state "wake up" will look:

Each of the four traits (title, desc, image, option), and even the traits for the options (label, link) can go in any order. Each of them is also optional and can be excluded if you want.

And finally, somewhere in the .js file (outside of any state[] = {};) you should declare which state is the starting state, like so:

current = "wake up";

Once you have this .js file with as many states as you want, you can use a link to the .js on the home page to begin playing. Happy adventuring!
