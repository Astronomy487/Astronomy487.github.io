Doc Template

A document formatting template and a tutorial to its use created by Astro with JS from Vrabbers. You are free to use and modify this template for any need.

Table of Contents

Pre-Body Elements

Head

Inside of the <head> of the document are four things:

  • A link to style.css
  • A link to a favicon
  • The title of the doument in <title>
  • A <style> or link to a stylesheet containing:
    • A font applied to <body> (ideally with bold and italic fontfaces)
    • A three-color palette which should look similar to this:
      • :root {
        --white: #f0f0f0;
        --color: #ff2e63;
        --black: #303841;
      }
    • Any other styling information you want to include yourself

If the font and colors are not supplied, the default values (as seen in this tutorial) will be used instead.

Page

The entire page is encased in a <div class="main"> within the <body>. This main div provides the massive margins and some default padding and margins. If you want to add your own elements outside of the main document (likely absolute or fixed elements, possibly for things like navigation), feel free to do so — just be sure it is outside the main div.

The only element outside of the main div in the template is a link to the JS that generates the table of contents.

Title

At the top of the page is the title of the document in an <h1>. Underneath it is a subtitle or short paragraph contained in a <div class="sub">, and it can be used for any information that you want to be seen immediately.

Table of Contents

The linked CSS and JS will automatically scan the document for headings and insert them into the table of contents. Including and excluding headings from the table of contents is explained later. This is the HTML for the table of contents:

<hr>
<h2>Table of Contents</h2>
<div id="toc"></div>
<hr>

If you use the prescribed template, all of the neccessary HTML will already be there for all the pre-body elements.

Body Elements

Headings

First order and second order headings are constructed with <h2> and <h3> tags, both of which require class="linkhead" in order to be automatically enumerated and entered into the table of contents. If you want to use an <h2> or <h3> without their enumeration, that will also work. Enumerated headings will break if any two have the same name.

Inline Elements

Every paragraph should be encased by <p> tags so that the spacing between them is consistent. Paragraphs are technically a block element, but are included among the inline elements because of the inline elements within paragraphs.

In paragraphs, you can use <b> and <a>, both of which look like this (although <a>s are underlined when hovered, like this). In addition, <i> can be used for italics like this, but be aware that not all fonts have built-in italic fontfaces. <mark> produces text like this. For all inline markups, be mindful of the scenarios you use them in and what they indicate to the reader — inconsistent use ruins the point of a well-formatted document.

Although I haven't written any custom CSS for these, <ul>, <ol>, <dl>, <dt>, and <dd> should all work as you'd expect in any HTML.

Block Elements

<table class="data"> creates a table fit for most purposes, within which you should put <tr>s for each row and, in turn, <td>s and <th>s inside those. There is also formatting for a <caption> before the first <tr>, but it is optional.

<code> creates a chunk of code. Within this snippet, however, you still have to use HTML like &lt; and &gt; instead of < and >, and <br> instead of a new line. They have no particular syntax highlighting. Code snippets looks like this:

<h3 class="linkhead">Block Elements</h3>

<blockquote>s are also supported, in addition to a <cite> within them. How you format these quotations — quotation marks, em dashes, citation formats — is up to you, but of course, stay consistent.

Figures and Media

Figures are the only element in the template that break the linear flow of the document. Instead of going after one block element and before another, they float to either the left or the right and allow text to wrap around it.

To include one, first identify where in the HTML it should be inserted. Text will begin to wrap immediately after — for this figure to the right, it is right after the Figures heading.

The class of the <figure> (either class="l" class="r") indicate which side of the page it will float to. Within the <figure> is an <img> and a <figcaption>, the former containing a URL and alt and the latter containing a caption. Either can be omitted and the figure will still display properly. Remember that your caption should not be the same as your alt text!

Figures default to 300px in width, and their images max out at 200px in height. The height of the figure will extend as far as neccessary for the caption.

Alternatively, if you want to include a block image or video between two other block elements, the <img> and <video> tags can be used. Their maximum dimensions are 500px by 400px. There is also bare-bones CSS for <audio>, although video and audio are not customized beyond appropriate margins and size. Overall, non-figure media is discouraged.

Using the Template

Obtaining and Editing

Because I host the CSS and JS files, the only file you have to manage is the HTML. An blank HTML template ready to be edited can be found here. Here is a full list of things you have to edit from this template yourself:
  • Change the <link rel="favicon" href=""> to include your favicon url under the href
  • Change the <title> to your title
  • Include a font and three-color palette in your <style> or, alternatively, reference a CSS file that contains this information (if this is excluded, the default palette and font will be used instead)
  • Change the <h1> to your title
  • Put a subtitle or short paragraph in <div class="sub"> (or delete it if you want)
  • Put all of the actual content (headings, inline elements, block elements, figures) in the spot marked with a comment in the template
